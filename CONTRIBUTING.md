# Contributing to Smart Electricity Bill Predictor & Analyzer

We welcome contributions from the community! By participating in this project, you agree to abide by our Code of Conduct.

## How to Contribute

There are many ways to contribute, from improving documentation to writing new features.

### 🐛 Reporting Bugs

If you find a bug, please open an issue on GitHub. Before submitting, please check if a similar issue already exists. When reporting a bug, include:

- A clear and concise description of the bug.
- Steps to reproduce the behavior.
- Expected behavior.
- Screenshots or error messages if applicable.
- Your operating system, browser, and Python/Node.js versions.

### ✨ Suggesting Enhancements

We love new ideas! If you have a suggestion for a new feature or an improvement, please open an issue. Describe your idea clearly and explain why it would be beneficial to the project.

### 💻 Code Contributions

1.  **Fork the repository**: Start by forking the project to your GitHub account.
2.  **Clone your fork**: `git clone https://github.com/YOUR_USERNAME/smart-electricity-bill-analyzer.git`
3.  **Create a new branch**: `git checkout -b feature/your-feature-name` or `bugfix/your-bugfix-name`.
4.  **Make your changes**: Implement your feature or fix the bug.
5.  **Write tests**: If applicable, add unit or integration tests for your changes.
6.  **Run tests**: Ensure all existing tests pass.
7.  **Format your code**: Follow the existing code style. For Python, consider `black` or `flake8`. For JavaScript, `prettier` or `eslint`.
8.  **Commit your changes**: Write clear and concise commit messages. Use a conventional commit style (e.g., `feat: add new feature`, `fix: resolve bug`).
9.  **Push to your fork**: `git push origin feature/your-feature-name`
10. **Open a Pull Request**:
    - Go to the original repository on GitHub.
    - Click "New pull request".
    - Provide a descriptive title and detailed explanation of your changes.
    - Reference any related issues.

## Code of Conduct
Please note that this project is released with a Code of Conduct. By participating in this project, you agree to abide by its terms.
